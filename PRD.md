# PRD: Telegram-бот недельного меню

## 1. Краткое описание

Telegram-бот помогает пользователю быстро получать меню на сегодня, завтра и всю неделю, видеть список покупок с ориентировочной стоимостью и учитывать продукты, которые уже есть дома. После команды `/start` бот сохраняет чат и ежедневно отправляет меню на текущий день в заданное время.

## 2. Цель продукта

Снизить ежедневные бытовые решения вокруг еды: что готовить, что купить и какие продукты уже есть дома.

Основная ценность:

- готовое недельное меню в Telegram;
- быстрый доступ к меню по командам;
- единый список покупок с примерной стоимостью;
- ежедневное напоминание с меню на день.

## 3. Целевая аудитория

Основной пользователь - человек или семья, которые планируют домашнее питание на неделю и закупаются в супермаркетах Португалии.

Контекст использования:

- утром нужно понять, что готовить сегодня;
- перед походом в магазин нужен список продуктов;
- в течение недели нужно быстро проверить блюда и остатки;
- меню и покупки удобнее получать в Telegram, без отдельного приложения.

## 4. Проблемы пользователя

- Меню на неделю хранится в заметках или файлах и его неудобно быстро открыть.
- Перед закупкой приходится вручную собирать продукты из рецептов.
- Сложно помнить, какие продукты уже есть дома.
- Ежедневно нужно заново решать, что готовить.

## 5. Пользовательские сценарии

### 5.1 Первичная активация

1. Пользователь открывает Telegram-бота.
2. Отправляет команду `/start`.
3. Бот сохраняет chat ID для ежедневной рассылки.
4. Бот сразу показывает меню на сегодня и список доступных команд.

### 5.2 Получение меню на сегодня

1. Пользователь отправляет `/today`.
2. Бот определяет текущий день недели по `BOT_TIMEZONE`.
3. Бот отправляет обед и ужин на текущий день.

### 5.3 Получение меню на завтра

1. Пользователь отправляет `/tomorrow`.
2. Бот определяет следующий день по `BOT_TIMEZONE`.
3. Бот отправляет меню на завтра.

### 5.4 Получение меню на неделю

1. Пользователь отправляет `/week`.
2. Бот отправляет меню на все 7 дней.

### 5.5 Получение списка покупок

1. Пользователь отправляет `/shopping`.
2. Бот отправляет список продуктов, количество, примерную цену и итоговый диапазон бюджета.
3. Бот показывает подсказку по магазинам: Pingo Doce, Mercadona, Auchan.

### 5.6 Проверка продуктов дома

1. Пользователь отправляет `/pantry`.
2. Бот отправляет список продуктов, которые уже есть дома.

### 5.7 Ежедневная рассылка

1. Пользователь ранее отправил `/start`.
2. Бот работает в фоне.
3. После наступления `DAILY_SEND_TIME` по `BOT_TIMEZONE` бот отправляет меню на сегодня.
4. За один календарный день рассылка отправляется не более одного раза.

## 6. Функциональные требования

### 6.1 Telegram-команды

Бот должен поддерживать команды:

- `/start` - сохранить чат для ежедневной рассылки и показать меню на сегодня;
- `/today` - показать меню на текущий день;
- `/tomorrow` - показать меню на следующий день;
- `/week` - показать меню на всю неделю;
- `/shopping` - показать список покупок и итоговую оценку стоимости;
- `/pantry` - показать список продуктов дома;
- `/help` - показать список команд.

### 6.2 Данные меню

Источник данных - файл `data/menu.json`.

Файл должен содержать:

- `week_menu` с ключами дней недели `monday` ... `sunday`;
- для каждого дня: `title`, `lunch`, `dinner`;
- `shopping_list` с названием, количеством и диапазоном цены;
- `pantry_items` со списком имеющихся продуктов;
- `recipes` с рецептами и шагами приготовления.

### 6.3 Формат сообщений

Сообщения должны отправляться в Telegram с HTML-разметкой.

Минимальные требования:

- заголовки выделяются жирным;
- меню дня содержит название дня, обед и ужин;
- список покупок содержит общий итог стоимости;
- неизвестная команда возвращает текст ошибки и справку.

### 6.4 Состояние бота

Бот должен хранить состояние в `bot_state.json`.

Состояние должно включать:

- `offset` для Telegram `getUpdates`;
- `chat_ids` для ежедневной рассылки;
- `last_daily_sent` для защиты от повторной отправки в один день.

### 6.5 Настройки окружения

Бот должен читать настройки из `.env`:

- `TELEGRAM_BOT_TOKEN` - обязательный токен Telegram-бота;
- `DAILY_SEND_TIME` - время ежедневной рассылки, по умолчанию `08:00`;
- `BOT_TIMEZONE` - часовой пояс, по умолчанию `Europe/Lisbon`.

Если `TELEGRAM_BOT_TOKEN` не задан, бот не должен запускаться и должен показать понятную ошибку.

## 7. Нефункциональные требования

### 7.1 Надежность

- При сетевых ошибках бот должен повторять попытку после паузы.
- Ошибка отправки ежедневного сообщения в один чат не должна останавливать отправку в другие чаты.
- Состояние должно сохраняться после обработки обновлений.

### 7.2 Простота эксплуатации

- Запуск должен выполняться одной командой: `python3 bot.py`.
- Основные настройки должны быть вынесены в `.env`.
- Данные меню должны редактироваться без изменения кода.

### 7.3 Локализация

- Основной язык интерфейса: русский.
- Часовой пояс по умолчанию: `Europe/Lisbon`.
- Цены отображаются в евро.

### 7.4 Безопасность

- Токен Telegram-бота не должен храниться в коде или в публичной документации.
- Файл `.env` должен оставаться локальным.
- Логи не должны раскрывать токен.

## 8. Ограничения и допущения

- Бот использует long polling через Telegram Bot API.
- Бот рассчитан на личное или семейное использование, а не на массовую аудиторию.
- Меню статичное и хранится локально в JSON.
- Редактирование меню через Telegram пока не входит в текущую версию.
- Автоматический расчет списка покупок из рецептов пока не реализован: список хранится явно.

## 9. Метрики успеха

- Пользователь может получить меню на сегодня за одно сообщение.
- Пользователь может получить список покупок перед закупкой за одно сообщение.
- Ежедневная рассылка приходит один раз в день в заданное время.
- Меню можно обновить через `data/menu.json` без правки Python-кода.
- Бот восстанавливается после временных сетевых ошибок без ручного вмешательства.

## 10. MVP

В MVP входят:

- запуск бота из локальной папки проекта;
- команды `/start`, `/today`, `/tomorrow`, `/week`, `/shopping`, `/pantry`, `/help`;
- чтение меню из `data/menu.json`;
- сохранение подписанных чатов в `bot_state.json`;
- ежедневная отправка меню в заданное время;
- базовая обработка сетевых ошибок.

## 11. Будущие улучшения

- Команда для просмотра рецептов, например `/recipes`.
- Команда для просмотра конкретного рецепта.
- Редактирование меню через Telegram.
- Генерация списка покупок на основе рецептов.
- Учет уже купленных продуктов.
- Поддержка нескольких разных недель меню.
- Разделение меню по пользователям или чатам.
- Dockerfile или systemd-инструкция для стабильного фонового запуска.
- Автоматические тесты для форматирования сообщений и логики ежедневной отправки.

## 12. Риски

- При ручном редактировании `data/menu.json` можно сломать формат файла.
- Если бот не запущен в момент ежедневной отправки, сообщение придет только после следующего запуска и проверки времени.
- При изменении часового пояса или времени рассылки нужно проверить, что `last_daily_sent` не блокирует нужную отправку.
- В `requirements.txt` указана зависимость `python-telegram-bot`, но текущая реализация использует прямые HTTP-запросы через стандартную библиотеку. Это стоит синхронизировать в будущей технической чистке.

## 13. Критерии приемки

- При `/start` бот сохраняет chat ID и возвращает меню на сегодня.
- При `/today` бот возвращает меню, соответствующее текущему дню недели.
- При `/tomorrow` бот возвращает меню следующего дня.
- При `/week` бот возвращает все 7 дней.
- При `/shopping` бот возвращает все позиции и корректно считает итоговый диапазон стоимости.
- При `/pantry` бот возвращает список имеющихся продуктов.
- При неизвестной команде бот возвращает справку.
- Ежедневная рассылка не дублируется в рамках одного календарного дня.
